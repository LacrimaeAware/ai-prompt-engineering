# Private Docs

This folder is for source-specific audit notes, local repo references, raw
handoffs after processing, and other context that should not be part of the
public instruction library.

Its contents are ignored by git except this README and `.gitkeep`.

Use this folder for:

- source maps naming local repositories or private paths
- raw handoff files from other models after they have been processed
- private examples with real names, private quotes, or local context
- repo-specific notes that should inform prompts but not appear in public docs

Public docs should contain distilled rules, examples, schemas, and workflows
that can stand without the private source context.
